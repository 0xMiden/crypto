Struct Subtree 

Source 
pub struct Subtree { /* private fields */ }
Expand description

Represents a complete 8-depth subtree that is serialized into a single RocksDB entry.

§What is stored

  • child_bits is a 512-bit bitmask (2 bits per node) marking non-empty left/right children.
  • hashes stores only non-empty child hashes in bit order.

§Local index layout (how indices are computed)

  • Indices are subtree-local and follow binary-heap (level-order) layout: root = 0; children of i are at 2i+1 and 2i+2.
  • Equivalently, given a (depth, value) from the parent tree, the local index is obtained by taking the node’s depth relative to the subtree root and its left-to-right position within that level (offset by the total number of nodes in all previous levels).

§In-memory format

  • child_bits: 8 little-endian u64s (64 bytes total).
  • hashes: contiguous Vec<Word> of non-empty child hashes.

pub fn len(&self) -> usize

Returns the number of non-empty inner nodes in this subtree.

A node “exists” if at least one of its two child bits is set. Each node occupies a 2-bit pair (left, right) in child_bits, so we collapse each pair into one bit and count. For a word w:

  1. w | (w >> 1) — propagates right-child bits into the even (left) position
  2. & 0x5555... — keeps only even positions (one bit per node)
  3. count_ones() — counts occupied nodes

Example with 4 nodes: w = 0b10_01_11_00

Node 0 (bits 0-1): left=0, right=0 → empty Node 1 (bits 2-3): left=1, right=1 → both children Node 2 (bits 4-5): left=1, right=0 → left only Node 3 (bits 6-7): left=0, right=1 → right only

  • w | (w >> 1) = 0b11_01_11_10 (OR each pair into even positions)
  • & 0x55 = 0b01_01_01_00 (mask to keep only even bits)
  • popcount = 3 (nodes 1, 2, 3 exist; node 0 is empty)

pub fn apply_mutations<'a>( &mut self, mutations: impl IntoIterator<Item = (&'a NodeIndex, &'a NodeMutation)>, )

Applies a batch of mutations to this subtree.

When mutations only update existing nodes (same structure, different hashes), hashes are patched in-place. Structural changes (additions or removals) trigger a rebuild.

  • NodeMutation::Addition(node) inserts or updates a node
  • NodeMutation::Removal removes a node

pub fn to_vec(&self) -> Vec<u8> ⓘ

Serializes this subtree into a compact byte representation.

The format is trivial since in-memory layout matches serialization:

  • 64 bytes: child_bits as little-endian u64s
  • Variable: non-empty child hashes (32 bytes each)
